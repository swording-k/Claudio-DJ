# Claudio DJ — 个人 AI 电台 设计文档

**版本**: 1.0
**日期**: 2026-04-24
**状态**: 已批准

---

## 一、产品定位

**Claudio DJ** 是一个跨平台（macOS / Windows / Android）的**主动触达型**个人 AI 电台助理。

核心逻辑：**了解你的口味 → 规划内容 → 像电台 DJ 一样播报**

不是问答工具，而是会主动唤醒你的"电台"——定时广播 + 随时语音唤醒，双轨并行。

---

## 二、系统架构（四层）

```
┌─────────────────────────────────────────────────────────────┐
│                    交互表层 (PWA + Tauri)                    │
│            Web 界面 / 语音唤醒 / 系统通知 / 媒体控制          │
├─────────────────────────────────────────────────────────────┤
│                    本地大脑 (Node.js 服务)                    │
│  ROUTER │ CONTEXT │ SCHEDULER │ TTS │ MUSIC │ STATE        │
├─────────────────────────────────────────────────────────────┤
│                    运行时聚合 (Prompt 组装)                   │
│  系统提示词 + 用户偏好 + 环境注入 + 检索记忆 + 输入/工具结果  │
├─────────────────────────────────────────────────────────────┤
│                    外部上下文 (API 集成)                      │
│  音乐(网易云) │ 语音(Fish Audio) │ 天气 │ 日历 │ AI模型     │
└─────────────────────────────────────────────────────────────┘
```

## 三、核心模块

| 模块 | 文件 | 职责 |
|------|------|------|
| router | `router.js` | 指令分发（语音/文字/定时触发） |
| context | `context.js` | 动态组装 AI 上下文窗口 |
| scheduler | `scheduler.js` | 定时任务调度（早/午/晚广播） |
| tts | `tts.js` | 文字转语音生成 |
| music | `music.js` | 网易云音乐搜索/获取URL/歌词 |
| state | `state.js` | 持久化（记忆 + 播报历史） |
| wakeword | Rust/Porcupine | 本地语音唤醒 |

## 四、AI 双轨制

### BYOK（自备 API Key）
用户配置自己的 API Key，我们负责路由和调度。

支持的模型：
- Claude (Anthropic)
- 通义千问 (阿里云)
- 豆包 (字节)
- 智谱 GLM

### 订阅制（代理 API）
- 适合不想折腾的技术小白
- 我们提供国内模型代理，按月收费

### 配置界面
```
config.json
{
  "model": "claude" | "qwen" | "doubao" | "glm",
  "apiKey": "user-provided-key",
  "subscription": {
    "enabled": false,
    "plan": "monthly"
  }
}
```

## 五、数据存储

位置：`~/.claudio/`

```
~/.claudio/
├── config.json       # API Keys、唤醒词、播报时间
├── taste.md          # 用户口味偏好（音乐类型、作息习惯）
├── routines.md       # 日常作息流程（早7点/午12点/晚10点）
├── memory/           # 向量检索记忆
│   └── *.md
├── state.db          # SQLite：播报历史、播放记录
└── cache/
    ├── tts/          # 语音缓存
    └── music/        # 音乐 URL 缓存
```

### 记忆文件格式

**taste.md**
```markdown
# 音乐口味
- 喜欢：华语流行、爵士、City Pop
- 不喜欢：太吵的重金属

# 生活习惯
- 早起时间：7:00
- 下班时间：18:30
- 睡前时间：23:00
```

**routines.md**
```markdown
# 定时播报
- 07:00 早间播报：天气 + 日历 + 推荐音乐
- 12:00 午间播报：午间资讯 + 音乐
- 21:00 晚间播报：总结 + 轻松音乐

# 特殊时刻
- 工作日 vs 周末 差异化处理
```

## 六、音乐播放流程

```
用户触发（语音/文字/定时）
        │
        ▼
AI 模型返回结构化 JSON：
{
  "say": "早上好，今天周三，给你放一首...",
  "play": ["歌曲名"],
  "segue": "接下来这首..."
}
        │
        ▼
music.js 调用 NeteaseCloudMusicApi
        │
        ▼
获取音乐 URL → 前端 <audio> 播放
        │
        ▼
可选：通过 UPnP 推送到客厅音响
```

### NeteaseCloudMusicApi 集成
- 项目：`NeteaseCloudMusicApi`
- 功能：搜索歌曲、获取播放 URL、歌词、推荐
- 部署：本地运行，端口 3000
- 限制：付费歌曲可能无法获取 URL（需用户开通会员）

## 七、交互模式

### 定时广播
- SCHEDULER.JS 按 routines.md 配置的时间触发
- 读取天气、日历、记忆
- 组装 Context → AI 生成播报 → TTS 语音播报
- 推送系统通知

### 语音唤醒
- 唤醒词：Porcupine 本地模型（可自定义训练）
- 唤醒后录音 → ASR → AI 处理 → 语音回复
- 支持"打断"：用户随时可以说"Hey Claudio"插话

### 文字交互
- PWA 界面打字输入
- AI 回复文字 + 可选语音播报
- 支持语音输入（按需录音，不持续监听）

## 十、UI/UX 设计

### 设计理念
这不是一个"内容浏览 App"，而是一个**声音控制台**——核心是听觉，不是视觉。UI 要做到：最小干扰、环境存在感、一触即达。

### 视觉语言：深夜电台 + 温暖科技

| 元素 | 选型 |
|------|------|
| 背景 | 深空黑 `#08080C`，接近纯黑但不刺眼 |
| 主色调 | 暖琥珀 `#F59E0B`，模拟真空管/胆机灯丝的温暖感 |
| 辅助色 | 柔白 `#E5E5E5`，文字和次要信息 |
| 强调色 | 霓虹蓝 `#3B82F6`，状态指示、激活状态 |

### 状态指示设计

| 状态 | 视觉表现 |
|------|---------|
| 待机 | 琥珀色圆点缓慢呼吸 |
| 正在播报 | 波形动画 + 进度条 |
| 语音监听中 | 麦克风图标闪烁 |
| 播放音乐 | 唱片旋转 / 音频可视化 |

### 交互原则

| 原则 | 说明 |
|------|------|
| 最小干扰 | 像电台一样在背景陪伴，不抢注意力 |
| 一触即达 | 播放/暂停/唤醒一键完成 |
| 感知状态 | 始终让用户知道 Claudio 在听/在说/在播放 |
| 夜间友好 | 深夜使用时灯光柔和不刺眼 |

### 动效规范

- 状态切换：200ms ease-out
- 波形动画：连续帧渲染，60fps 目标
- 呼吸灯效果：2s 周期，opacity 0.4-1.0
- 禁用花哨的粒子效果
